# VoiceKey Open Source Governance Specification

> Version: 1.0
> Date: 2026-02-19

---

## 1. Required Repository Files

- `LICENSE`
- `README.md`
- `CONTRIBUTING.md`
- `CODE_OF_CONDUCT.md`
- `SECURITY.md`
- `.github/ISSUE_TEMPLATE/*`
- `.github/PULL_REQUEST_TEMPLATE.md`

---

## 2. Licensing Policy

- Project license: MIT or Apache-2.0 (must be explicitly chosen and documented)
- Dependency license compatibility review required before adding new runtime dependency

---

## 3. Contribution Workflow

1. open issue (bug/feature/proposal)
2. submit PR linked to issue
3. pass required CI checks
4. maintainer review and merge

---

## 4. Versioning and Changelog

- semantic versioning (`MAJOR.MINOR.PATCH`)
- human-readable changelog per release
- breaking changes must include migration guidance

---

## 5. Security Disclosure

- responsible disclosure policy in `SECURITY.md`
- private contact channel for vulnerability reports
- defined SLA for acknowledgement and remediation updates

---

## 6. Community Health Targets

- issue response within defined target window
- contributor documentation kept current
- release cadence published publicly
